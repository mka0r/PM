Часть 1. Идентификация проблем


Приоритизированный по влиянию на сроки и бюджет список проблем:

1. Критическая техническая нестабильность: База данных приложения на iOS не выдерживает минимальной нагрузки, что делает невозможным его использование и выпуск для реальных пользователей, несмотря на готовность любых функций.

2. Внутренние блокировки команды: Андроид-разработчик простаивает и не может выполнять свои задачи из-за неготовности API со стороны бэкенд-разработчика, что приводит к прямым потерям времени и бюджета.

3. Отсутствие единого приоритета и постоянное расширение границ проекта: Команда получает противоречивые указания от СЕО, который одновременно требует срочной монетизации и добавляет новые функции (редизайн иконки, раздел с рецептами), что распыляет фокус и саботирует достижение главной цели, также в Trello висит размытая задача «Тестирование».

4. Отсутствие четких требований и объемов: Дизайнер и разработчики не могут завершить работу над разделом "Тренировки", так как не определен и не согласован базовый контент (упражнения), что ведет к использованию заглушек и необходимости последующих переделок.

5. Невыровненные ожидания со стейкхолдерами: Маркетолог планирует рекламную активность к выставке через 2 месяца и готовит материалы к дедлайну, не располагая информацией о реальном состоянии проекта и технических рисках, что создает почву для репутационных потерь и конфликтов.









Часть 2. Выбор методологии и первые шаги

2.1

Я предлагаю внедрить методологию Канбан, используя для этого Jira как единую платформу для управления задачами.

Что мы сделаем в Jira:

Создадим Канбан-доску для проекта "Фитнес-Гуру".
Внесём все текущие задачи (от "Исправить падение БД" до "Определить контент для тренировок").
Назначим ответственных, разобьем на подзадачи, выставим приоритеты и будем отслеживать прогресс в реальном времени.

Эта методология идеально подходит для нашей ситуации, потому что она:

Немедленно визуализирует хаос и блокировки (падение БД, отсутствие API и т. д.), позволяя команде сконцентрироваться на их устранении.
Начинает наводить порядок без перегрузки команды строгими правилами, что критично при демотивации и отсутствии процессов.
Создаёт прозрачность для стейкхолдеров в Jira, помогая управлять их ожиданиями на основе реальной картины, а не предположений.

Совместно фиксируем в Jira 3 самые критичные задачи, которые нужно закрыть в первую очередь:
•	Задача 1: "Исправить падение базы данных на тестовом стенде"
o	Обсуждаем: как воспроизводится ошибка? Какие логи есть? Кто (iOS-разработчик/бэкенд) будет ответственным за решение? Какие ресурсы нужны?
o	Цель: занести в Jira с назначением ответственного и четкими критериями приемки ("БД стабильна при N одновременных подключениях").
•	Задача 2: "Завершить и передать API для авторизации"
o	Обсуждаем: В каком состоянии API сейчас? Что конкретно не хватает Android-разработчику? Уточняем сроки сдачи от бэкенд-разработчика.
o	Цель: зафиксировать в Jira конечные точки API и дедлайн для бэкенда, снять блокировку с Android-разработчика.
•	Задача 3: "Определить и согласовать контент для раздела 'Тренировки'"
o	Обсуждаем: Кто отвечает за контент (СЕО, приглашенный эксперт)? Какой минимальный список упражнений нужен для запуска? Сроки предоставления данных.
o	Цель: назначить ответственного за предоставление контента и срок, чтобы дизайнер и разработчики могли убрать заглушки.
1.	Резюме и следующие шаги (5 минут)

Обоснование плана
Почему такая структура? План построен по логике разработки сложных фич с акцентом на безопасность платежей. Задачи анализа и проектирования вынесены в начало, чтобы избежать дорогих переделок. Увеличенное время на тестирование обеспечит надежность финансовых операций.
Основные риски:
•	Задержки согласований с платежными провайдерами
•	Длительная модерация в App Store и Google Play
•	Непредвиденные сложности при интеграции
Ключевые допущения:
•	Команда сфокусирована только на этой задаче
•	Критические технические долги устранены до начала работ
•	Платежный шлюз выбран в течение первой недели
Общая продолжительность: 55 рабочих дней (~11 недель)
